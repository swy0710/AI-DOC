# tmux 완벽 가이드 — AI 에이전트 시대의 터미널 멀티플렉서

## 1. tmux란?

2007년 니콜라스 메리가 개발한 터미널 멀티플렉서로, 약 19년의 역사를 가지고 있다. 최근 Claude Code의 에이전트 팀 기능이 등장하면서 다시 주목받고 있다.

**터미널 에뮬레이터 vs 터미널 멀티플렉서**의 핵심 차이는 세션 유지 여부다. 터미널 에뮬레이터(iTerm, Ghostty, Warp 등)는 앱을 닫으면 쉘 세션이 함께 종료되지만, tmux는 서버가 쉘을 관리하기 때문에 터미널을 닫아도 세션이 그대로 유지된다. `tmux attach`로 언제든 복귀 가능.

추천 터미널 에뮬레이터는 **Ghostty** — GPU 가속으로 빠르고, CPU/메모리 사용량이 적다. tmux가 탭/분할/세션을 담당하므로 에뮬레이터는 가볍고 빠른 게 핵심.

---

## 2. 3계층 구조

```
Session (세션)
 └── Window (윈도우)
      └── Pane (페인)
```

세션은 여러 윈도우를 포함하고, 윈도우는 여러 페인으로 분할된다.

---

## 3. 핵심 단축키

모든 명령은 **Prefix** (기본: `Ctrl + B`) 입력 후 실행한다.

### 윈도우

| 단축키 | 동작 |
| --- | --- |
| `Prefix + c` | 새 윈도우 생성 |
| `Prefix + p` / `Prefix + n` | 이전/다음 윈도우 |
| `Prefix + [0-9]` | 인덱스로 윈도우 직접 이동 |
| `Prefix + ,` | 윈도우 이름 변경 |

### 페인

| 단축키 | 동작 |
| --- | --- |
| `Prefix + %` | 수평 분할 (좌우) |
| `Prefix + "` | 수직 분할 (상하) |
| `Prefix + 방향키` | 페인 간 이동 |
| `Prefix + z` | 페인 줌 (최대화/복구 토글) |

### 세션

| 단축키 | 동작 |
| --- | --- |
| `Prefix + d` | 세션 디태치 |
| `Prefix + s` | 세션 목록 |
| `Prefix + w` | 윈도우 리스트 (세션 포함, 더 실용적) |
| `Prefix + $` | 세션 이름 변경 |
| `Prefix + :new` | tmux 내부에서 새 세션 생성 |

### 터미널에서

| 명령어 | 동작 |
| --- | --- |
| `tmux` / `tmux new` | 새 세션 생성 |
| `tmux ls` | 세션 목록 확인 |
| `tmux attach` / `tmux a` | 세션 재연결 |

## 4. 추천 설정 (`~/.tmux.conf`)

**Prefix 변경** — `Ctrl + B` → `Ctrl + Space`로 변경하면 손이 편하다.

**히스토리 리밋** — 기본 2,000줄을 50,000줄로 늘려서 개발 서버 로그를 충분히 보존.

**마우스 활성화** (`set -g mouse on`) — 페인 클릭 이동, 창 너비 드래그 조절 가능.

**Vi 키 바인딩** — 페인 이동을 `h/j/k/l`로 매핑. `Option + h/j/k/l` 조합으로 Prefix 없이 이동하도록 설정하면 더 빠르다. Vi 방향키는 원격 서버 작업에서도 활용도가 높으니 익혀두는 것을 권장.

**테마** — Catppuccin 테마 적용. 상태바 좌측에 세션/윈도우 이름, 우측에 현재 디렉토리, Git 브랜치, CPU, 메모리, 배터리, 시간 표시.

**Yazi 파일 탐색기 연동** — `Prefix + Tab`으로 화면 분할하여 Yazi 실행.

## 5. 스크립팅을 통한 개발 환경 자동화

tmux의 진짜 강점은 **프로그래머틱하게 조작 가능**하다는 점이다.

### 핵심 스크립트 명령어

```bash
# 특정 페인에 명령어 전달
tmux send-keys -t [target] '명령어' Enter

# 특정 페인의 히스토리(로그) 읽어오기
tmux capture-pane -t [target] -p
```

### 활용 예시

여러 프로젝트의 세션/윈도우/페인을 나누고, 개발 서버 + Claude Code를 한 번의 스크립트 실행으로 전부 시작. 백엔드 세션, 프론트엔드 세션, Claude Code 세션을 자동으로 구성하여 13개 이상의 윈도우를 한 번에 띄울 수 있다.

## 6. AI 에이전트 연동 — Cross Verify (교차 검증)

tmux 스크립팅 + AI 에이전트의 조합으로 가능한 워크플로우:

**기본 활용**: 개발 서버 에러 로그를 `capture-pane`으로 가져와서 Claude Code에게 요약 요청. 현재 세션뿐 아니라 다른 세션의 윈도우 로그까지 가져와 분석 가능.

### 교차 검증 스킬 구조

```
Claude Code (팀 리드)
 ├── Codex CLI → Codex 리뷰어 에이전트 (별도 페인)
 └── Gemini CLI → Gemini 리뷰어 에이전트 (별도 페인)
```

- 동일 주제에 대해 서로 다른 모델이 독립적으로 검증
- 팀 리드가 결과를 종합 분석하여 최종 결과물 생성
- 단일 에이전트의 컨텍스트 윈도우 한계를 넘어서, 서로 다른 모델의 전문화된 인사이트를 종합하여 더 정확한 결과를 얻을 수 있다

---

## 7. 핵심 요약

tmux는 **세션 유지**, **3계층 구조(세션/윈도우/페인)**, **스크립팅 자동화**라는 세 가지 축으로 동작한다. 특히 멀티 에이전트를 병렬로 관리하는 개발 방식이 확산되면서, 에이전트를 프로그래머틱하게 스폰하고 제어할 수 있는 tmux의 가치가 극대화되고 있다.
